## The Problem We Do Not Talk About Enough

Every day, millions of wearable devices collect billions of health data points. This data promises to revolutionize preventive care, enable early disease detection, and personalize medicine. But not everyone is represented in this data. And for those who are, the data itself may not be equally reliable.

<div align="center">

| Disparity | Finding |
|-----------|---------|
| **Ownership Gap** | 50% difference between highest and lowest income brackets |
| **Sensor Accuracy** | 11% gap in accuracy by skin tone category |
| **Fragmentation** | Lower-income users face 2.5x higher data fragmentation |
| **Clinical Integration** | Only 41% of health-focused users share data with providers |

</div>

## Data Architecture

The toolkit uses a three-bucket data model designed to capture the full pathway from potential user to clinically integrated data:

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   BUCKET 1                    BUCKET 2                    BUCKET 3          │
│   Population & Usage          Device & Fragmentation      Quality & Bias    │
│                                                                             │
│   ┌─────────────────┐        ┌─────────────────┐        ┌─────────────────┐ │
│   │ Demographics    │        │ Brand & Model   │        │ Sensor Accuracy │ │
│   │ Income, Race    │───────▶│ Data Format     │───────▶│ Skin Tone       │ │
│   │ Geography       │        │ API Access      │        │ Error Rates     │ │
│   │ Insurance       │        │ EHR Export      │        │ Dropout Rates   │ │
│   │                 │        │ Fragmentation   │        │ Wear Time       │ │
│   │ owns_wearable   │        │ Index (0-1)     │        │ Sharing         │ │
│   │ uses_for_health │        │                 │        │ Behavior        │ │
│   │ shares_data     │        │                 │        │                 │ │
│   └─────────────────┘        └─────────────────┘        └─────────────────┘ │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## Quick Start

### Installation

```bash
pip install pandas numpy scikit-learn plotly kaleido
```

### Run Analysis

```bash
# Clone the repository
git clone https://github.com/Cazzy-Aporbo/Wearable-Health-Data.git
cd Wearable-Health-Data

# Run with synthetic data
python main.py --scenario baseline --n-users 2000 --output results
```

### Programmatic Usage

```python
from main import run_full_analysis

results = run_full_analysis(
    scenario_name="baseline",      # or: high_disparity, improved_equity, fragmented_ecosystem
    n_users=2000,
    output_dir="my_results"
)

print(f"Ownership gap: {results['data_summary']['ownership_rate']:.1%}")
```

## Research Context

This toolkit addresses themes documented across digital health equity research:

**Digital Divide Patterns**: Wearable device ownership varies substantially by socioeconomic status, creating systematic gaps in who contributes data to health algorithms.

**Sensor Accuracy Variations**: Photoplethysmography sensors may produce less accurate readings for individuals with darker skin tones, with implications for clinical reliability.

**Data Fragmentation**: The wearable ecosystem remains fragmented across proprietary platforms, limiting interoperability and clinical integration potential.

**Representativity Concerns**: Datasets used to train health algorithms may systematically underrepresent certain populations, risking biased outputs.

## Disclaimer

This toolkit is intended for research and exploratory purposes only. It does not perform clinical diagnosis or provide personalized medical advice. Synthetic data captures directional patterns but should not be interpreted as precise population-level estimates. Findings should be validated with domain expertise and replicated with real-world data from relevant populations.
